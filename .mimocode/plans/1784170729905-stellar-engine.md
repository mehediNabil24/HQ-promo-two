# Plan: Process Steps — Move active state on hover

## Problem
The first process card has `process__step--active` by default (hardcoded in HTML). When hovering any other card, the first card's active state doesn't reset — it stays active alongside the hovered card.

## Solution
Add a `mouseenter` event listener on each `.process__step` that:
1. Removes `process__step--active` from ALL steps
2. Adds `process__step--active` to the hovered step

## File to modify
- `js/main.js` — add at end of IIFE (before closing `})()`)
  ```js
  const processSteps = document.querySelectorAll(".process__step");
  processSteps.forEach((step) => {
    step.addEventListener("mouseenter", () => {
      processSteps.forEach((s) => s.classList.remove("process__step--active"));
      step.classList.add("process__step--active");
    });
  });
  ```

## No CSS changes needed
The existing CSS already uses `:hover` and `.process__step--active` selectors for all visual states (card bg, title color, text color, icon, pill text, polygon).

## Verification
- Open index.html
- First card starts active (orange gradient bg, white text)
- Hover second card → first card resets to dark bg, second card gets orange gradient
- Hover third card → second resets, third activates
- Mouse leaves all cards → last hovered card stays active (current behavior)
